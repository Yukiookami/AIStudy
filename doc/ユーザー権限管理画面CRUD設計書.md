# ユーザー権限管理画面 CRUD 設計書

## 1. 画面目的

本画面は、ユーザーごとのロール・権限情報を管理するための画面である。

学習用デモのため、実際の認証処理、API 通信、DB 連携は行わない。  
画面内の state でデータを保持し、CRUD 操作を確認できる構成とする。

---

## 2. 実装対象

| 項目           | 内容                   |
| -------------- | ---------------------- |
| 画面名         | ユーザー権限管理画面   |
| 対象 URL       | `/dashboard`           |
| 対象画面       | 既存の `DashboardPage` |
| 実装方針       | 既存画面を修正する     |
| 新規ページ作成 | 不要                   |

---

## 3. 画面構成

画面は以下の構成とする。

| エリア         | 内容                                           |
| -------------- | ---------------------------------------------- |
| ヘッダーエリア | 画面タイトルと説明文を表示する                 |
| サマリーエリア | ユーザー数、ロール数、有効ユーザー数を表示する |
| 検索エリア     | キーワード、状態による検索条件を表示する       |
| 一覧エリア     | ユーザー権限情報を一覧表示する                 |
| 操作エリア     | 新規追加、編集、削除の操作を行う               |
| フォームエリア | 新規追加・編集用フォームを表示する             |

---

## 4. 初期表示

画面表示時、以下を行う。

1. ユーザー権限一覧を表示する。
2. サマリー情報を表示する。
3. 検索条件は未入力状態とする。
4. 一覧にはすべての初期データを表示する。

---

## 5. データ定義

### 5.1 ユーザー権限データ項目

| 項目名     | 型     | 必須 | 説明           |
| ---------- | ------ | ---- | -------------- |
| id         | number | 必須 | ユーザーID     |
| name       | string | 必須 | ユーザー名     |
| email      | string | 必須 | メールアドレス |
| role       | string | 必須 | ロール         |
| department | string | 必須 | 所属部署       |
| status     | string | 必須 | 状態           |
| note       | string | 任意 | 備考           |

### 5.2 状態

| 値      | 表示内容 |
| ------- | -------- |
| active  | 有効     |
| stopped | 停止中   |

### 5.3 ロール

| 値     | 表示内容 |
| ------ | -------- |
| admin  | 管理者   |
| editor | 編集者   |
| viewer | 閲覧者   |

## 7. サマリー表示

サマリーエリアには以下を表示する。

| 表示項目       | 内容                       |
| -------------- | -------------------------- |
| ユーザー数     | 現在の一覧データ件数       |
| ロール数       | 利用中のロール種類数       |
| 有効ユーザー数 | 状態が「有効」のユーザー数 |

検索中の場合も、サマリーは現在保持している全データを基準に表示する。

---

## 8. 検索機能

### 8.1 検索条件

検索エリアには以下を表示する。

| 項目       | 入力形式     | 説明                                                           |
| ---------- | ------------ | -------------------------------------------------------------- |
| キーワード | テキスト入力 | ユーザー名、メールアドレス、部署、備考を対象に部分一致検索する |
| 状態       | セレクト     | すべて、有効、停止中から選択する                               |

### 8.2 検索ボタン

検索ボタンのラベルは以下とする。

| 項目     | 内容     |
| -------- | -------- |
| ボタン名 | 検索開始 |

### 8.3 検索処理

検索開始ボタンを押下すると、入力された検索条件に一致するデータのみ一覧に表示する。

キーワード検索は以下の項目を対象とする。

- ユーザー名
- メールアドレス
- 部署
- 備考

### 8.4 リセット処理

リセットボタンを押下すると、検索条件を初期化し、全件表示に戻す。

---

## 9. 一覧表示

### 9.1 表示項目

一覧には以下を表示する。

| 表示項目       | 内容                       |
| -------------- | -------------------------- |
| ユーザー名     | ユーザー名を表示する       |
| メールアドレス | メールアドレスを表示する   |
| ロール         | ロールを表示する           |
| 部署           | 所属部署を表示する         |
| 状態           | 有効 / 停止中 を表示する   |
| 備考           | 備考を表示する             |
| 操作           | 編集、削除ボタンを表示する |

### 9.2 状態表示

状態は視認しやすい形式で表示する。

| 状態   | 表示方針               |
| ------ | ---------------------- |
| 有効   | 正常状態として表示する |
| 停止中 | 注意状態として表示する |

---

## 10. 新規追加機能

### 10.1 新規追加開始

新規追加ボタンを押下すると、登録フォームを表示する。

### 10.2 新規追加フォーム

登録フォームには以下の項目を表示する。

| 項目           | 必須 | 入力条件                       |
| -------------- | ---- | ------------------------------ |
| ユーザー名     | 必須 | 1〜30文字                      |
| メールアドレス | 必須 | メール形式                     |
| ロール         | 必須 | 管理者、編集者、閲覧者から選択 |
| 部署           | 必須 | 1〜30文字                      |
| 状態           | 必須 | 有効、停止中から選択           |
| 備考           | 任意 | 100文字以内                    |

### 10.3 登録処理

登録ボタンを押下すると、入力内容を一覧データに追加する。

登録時の ID は、既存データの最大 ID + 1 とする。

登録完了後、以下を行う。

1. フォームを閉じる。
2. 一覧を更新する。
3. 入力内容を初期化する。

---

## 11. 編集機能

### 11.1 編集開始

一覧の編集ボタンを押下すると、対象データをフォームに表示する。

### 11.2 編集フォーム

編集フォームの入力項目は、新規追加フォームと同じとする。

### 11.3 更新処理

保存ボタンを押下すると、対象データを入力内容で更新する。

更新完了後、以下を行う。

1. フォームを閉じる。
2. 一覧を更新する。
3. 編集対象を初期化する。

---

## 12. 削除機能

### 12.1 削除開始

一覧の削除ボタンを押下すると、削除確認ダイアログを表示する。

### 12.2 削除確認

削除確認ダイアログには、削除対象のユーザー名を表示する。

### 12.3 削除処理

確認後、対象データを一覧から削除する。

削除をキャンセルした場合、データは変更しない。

---

## 13. バリデーション

### 13.1 必須チェック

以下の項目は必須とする。

| 項目           | エラーメッセージ                 |
| -------------- | -------------------------------- |
| ユーザー名     | ユーザー名を入力してください     |
| メールアドレス | メールアドレスを入力してください |
| ロール         | ロールを選択してください         |
| 部署           | 部署を入力してください           |
| 状態           | 状態を選択してください           |

### 13.2 文字数チェック

| 項目       | 条件        | エラーメッセージ                         |
| ---------- | ----------- | ---------------------------------------- |
| ユーザー名 | 30文字以内  | ユーザー名は30文字以内で入力してください |
| 部署       | 30文字以内  | 部署は30文字以内で入力してください       |
| 備考       | 100文字以内 | 備考は100文字以内で入力してください      |

### 13.3 入力形式チェック

| 項目           | 条件       | エラーメッセージ                       |
| -------------- | ---------- | -------------------------------------- |
| メールアドレス | メール形式 | メールアドレスの形式で入力してください |

---

## 14. ボタン一覧

| ボタン名   | 表示位置         | 処理内容                           |
| ---------- | ---------------- | ---------------------------------- |
| 検索開始   | 検索エリア       | 検索条件に一致するデータを表示する |
| リセット   | 検索エリア       | 検索条件を初期化する               |
| 新規追加   | 一覧エリア上部   | 新規追加フォームを表示する         |
| 編集       | 一覧の各行       | 対象データを編集フォームに表示する |
| 削除       | 一覧の各行       | 削除確認ダイアログを表示する       |
| 登録       | 新規追加フォーム | 入力内容を一覧に追加する           |
| 保存       | 編集フォーム     | 対象データを更新する               |
| キャンセル | フォーム         | フォームを閉じる                   |

---

## 15. 画面遷移

| 操作               | 遷移先       |
| ------------------ | ------------ |
| ログイン画面へ戻る | `/`          |
| ダッシュボード表示 | `/dashboard` |

CRUD 操作による別画面への遷移は行わない。

---

## 16. 対象外

以下は今回の実装対象外とする。

- 実際のログイン認証
- API 通信
- DB 連携
- 権限による表示制御
- ロールによる操作制御
- 状態管理ライブラリの追加
- ページネーション
- ソート
- CSV 出力
- ファイルアップロード

## 18. 完了条件

以下を満たした場合、実装完了とする。

- ユーザー権限一覧が表示されること
- キーワード検索ができること
- 状態検索ができること
- 検索条件をリセットできること
- 新規追加ができること
- 編集ができること
- 削除ができること
- 必須チェックが動作すること
- 文字数チェックが動作すること
- メール形式チェックが動作すること
- ログイン画面へ戻れること
- 不要な機能が追加されていないこと
