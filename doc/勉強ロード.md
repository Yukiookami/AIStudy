# GitHub Copilot

## 環境

VS Code に GitHub Copilot 拡張機能をインストールし、VS Code 環境で GitHub Copilot を学習する。

### VS Code を利用する理由

- GitHub Copilot の最新機能が最も早く提供される
- 公式ドキュメントやサンプルの多くが VS Code を前提としている
- 拡張機能をインストールするだけで利用できる

### VS Code のインストール

ブラウザで「Visual Studio Code」を検索し、公式サイトからダウンロード・インストールする。

# 学習ロード

## 1. 基本操作

### 内容

#### Copilot Chat

- Ask モード
  - GPT のように質問へ回答する。
  - ファイルの編集は行わない。

- Plan モード
  - タスクを分析し、実施内容をプランとして作成する。
  - プランを確認した後、Agent モードで実行できる。

- ~~Edit モード~~
  - ファイルを編集する。（最新バージョンではAgentモードとして合併しました）

- Agent モード
  - プラン作成を含め、複数ファイルを横断して修正・作成を行う。

#### その他

- インラインチャット
- コード補完

### 目標

- コード生成
- コード説明
- バグ修正
- 基本機能を使いこなせるようになる

---

## 2. Context

### 内容

- `#file`
  - 特定のファイルを明示して、その内容を前提に質問・依頼する。
- `#selection`
  - 現在選択している範囲だけを対象に、ピンポイントで質問・編集を依頼する。
- `#workspace`
  - ワークスペース全体を対象に、関連ファイルを横断して調べてもらう。
- `#terminal`
  - ターミナルの実行結果やログをコンテキストとして渡す。
- `#problems`
  - Problems ビューのエラーや警告を共有し、問題箇所を前提に修正相談する。
- `#changes`
  - 変更済みファイルや差分を共有し、どこを修正したかを踏まえて会話する。

### 目標（Context）

Copilot が「どの情報を見て回答しているか」を理解し、必要なコンテキストだけを渡せるようになる。
